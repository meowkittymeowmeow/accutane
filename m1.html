module 1 — core ideas

1. population vs sample

  • population: everything you ultimately care about (could be huge or even hypothetical).
  • sample: the part you actually observe.
  • probability assumes the population is known and asks what samples should look like.
  • statistics has only the sample and asks what can be inferred about the population.

2. what statistics is

  • using data to learn something non-trivial.
  • “ethical” = using the method only when its assumptions actually hold.

3. why probability comes first

  • all inference is: “if the world behaved like X, how likely is data like this?”
  • you need the machinery of randomness before flipping that logic around.

4. experiments, sample spaces, events

  • experiment: process that yields an observable outcome.
  • sample space S: set of all possible outcomes.
  • event: any subset of S.
  • cardinality: number of elements.

5. examples

  • one coin flip: S = {0,1}. |S| = 2.
  • two flips: S = {00,01,10,11}. |S| = 4.
  • flip until first tail: S = {T, HT, HHT, HHHT, …} (infinite).
  • car defects (3 binary features): total 2³ = 8 outcomes.

6. set notation

  • A ∪ B: outcomes in A or B.
  • A ∩ B: outcomes in both.
  • Aᶜ: outcomes in S but not in A.
  • disjoint: A ∩ B = ∅.

7. car-defect event example

  • A: engine problem = {100,110,101,111}
  • B: exactly one defect = {100,010,001}
  • C: exactly two defects = {110,101,011}

derived:

  • A ∩ B = {100}
  • Aᶜ = {000,010,001}
  • Aᶜ ∪ B = {000,010,001,100}
  • B ∩ C = ∅

8. venn diagrams

pure bookkeeping; each outcome lives in exactly one region.
