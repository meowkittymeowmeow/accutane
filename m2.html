module 1 — counting

setup: you have n distinct options (e.g. n balls, n students, n symbols) and you choose k positions (e.g. 3 draws, 4 seats, 2 slots in a code).

two binary choices:

  • ordered vs unordered: does AB differ from BA?
  • with vs without replacement: can you reuse the same option more than once?

1. ordered, with replacement → nk

each position is filled independently, you can reuse the same item.

  • slot 1: n choices
  • slot 2: n choices
  • …
  • slot k: n choices

by multiplication: total = n × n × … × n (k times) = nk.

example: 10-digit code, digits 0–9 allowed each time → 1010 possibilities.

2. ordered, without replacement → n! / (n − k)!

now once you use an item, it’s gone.

  • slot 1: n choices
  • slot 2: n − 1 choices
  • slot 3: n − 2 choices
  • …
  • slot k: n − k + 1 choices

total = n × (n − 1) × … × (n − k + 1) = n! / (n − k)! (a k-permutation of n).

example: 5 students, choose a president, VP, and treasurer → 5 × 4 × 3 = 60.

3. unordered, without replacement → C(n, k)

you only care which items are chosen, not their order. every group of k items is a “committee”.

start from the ordered-without count (permutations) and then collapse all k! reorderings into one:

  • ordered, without replacement: n! / (n − k)!
  • each unordered set corresponds to k! orderings

so number of distinct groups =

C(n, k) = n! / (k! (n − k)!).

example: 5 students, choose any 3 to be on a committee, all equal → C(5, 3) = 10.

4. unordered, with replacement → C(n + k − 1, k)

you pick k items, allowed to reuse, and only the multiset matters (counts of each option).

classic “stars and bars” result: number of ways to distribute k identical balls into n distinct boxes =

C(n + k − 1, k).

interpretation: you’re choosing where to place k stars and (n − 1) dividers on a line; positions chosen for stars determine the multiset.
